     Fix: Bar chart data binning when compacted

     Context

     When the bar chart is in its default (non-zoomed/compacted) state, Vico
     only displays the first ~5 hours of data instead of fitting all data into
     the visible area. The user wants the chart to properly bin/aggregate data
     so all time periods are visible and meaningful when compacted.

     Files to modify

     - app/src/main/java/com/example/octopusdashboard/ui/chart/PriceLineChart.kt
     — main chart component
     -
     app/src/main/java/com/example/octopusdashboard/ui/chart/ChartComponents.kt
     — add binning utility

     Implementation

     1. Add data binning utility to ChartComponents.kt

     Create a BinnedPoint data class and a binPoints() function that aggregates
     HalfHourPoint lists into time-based bins:

     data class BinnedPoint(
         val intervalStart: Instant,
         val intervalEnd: Instant,
         val avgPrice: Double?,
         val totalConsumption: Double?,
         val totalCost: Double?
     )

     Binning strategy based on number of data points:

     ┌───────────────────┬──────────┬─────────────────────────────────┐
     │      Points       │ Bin Size │           Aggregation           │
     ├───────────────────┼──────────┼─────────────────────────────────┤
     │ ≤ 48 (1 day)      │ None     │ Show raw 30-min points          │
     ├───────────────────┼──────────┼─────────────────────────────────┤
     │ ≤ 168 (3.5 days)  │ 2 hours  │ Avg price, sum consumption/cost │
     ├───────────────────┼──────────┼─────────────────────────────────┤
     │ ≤ 720 (15 days)   │ 6 hours  │ Avg price, sum consumption/cost │
     ├───────────────────┼──────────┼─────────────────────────────────┤
     │ ≤ 2000 (~6 weeks) │ 1 day    │ Avg price, sum consumption/cost │
     ├───────────────────┼──────────┼─────────────────────────────────┤
     │ > 2000            │ 3 days   │ Avg price, sum consumption/cost │
     └───────────────────┴──────────┴─────────────────────────────────┘

     The function groups points by intervalStart truncated to the bin boundary,
     then computes averages for price and sums for consumption/cost.

     2. Update PriceLineChart.kt — chart rendering

     - Import Zoom from Vico
     - When isZoomed = false: bin the data before feeding to the model producer,
     and use Zoom.Content as initialZoom to ensure the chart fits all binned
     data
     - When isZoomed = true: use raw data (no binning) with scroll/zoom enabled
     - Update the LaunchedEffect to use binned or raw data based on isZoomed
     state
     - Update the time axis formatter to work with binned points (show bin start
     time)
     - Update marker value formatter to show appropriate values for binned data
     (avg price label, total consumption label)

     3. Key changes in PriceLineChart.kt

     // Compute binned data when not zoomed

         if (isZoomed) points else binPoints(points)
     }

     // Use binned indices for chart

     // Zoom state with Zoom.Content for compact mode

         zoomEnabled = isZoomed,

     Zoom.Content

     2. Select "7D" in compact mode — all 7 days should be visible with ~4-hour

     3. Select "1M" in compact mode — all 30 days visible with daily bins
